## 这篇指南解决什么

GitHub（代码和文档托管平台）可以用来公开分享这套资料包，也可以当作版本备份。对普通用户来说，最重要的不是命令，而是两件事：

1. 上传前把私人信息删干净。
2. 上传后别人下载就能直接用 Obsidian 打开。

如果只是自己用，不一定要上传 GitHub。用本地备份、网盘备份也可以。

## 公开前先分清两套库

建议长期保留两份：

- 公开库：模板、方法、脱敏案例、操作手册。
- 私人库：真实姓名、学校、成绩、证件、聊天记录、申请材料、个人复盘。

公开库可以上传 GitHub。私人库不要公开。

## 脱敏检查清单

上传前至少检查这些内容：

- 是否有真实姓名、手机号、邮箱、身份证号、学号。
- 是否有完整成绩单、排名截图、录取材料。
- 是否有聊天记录、导师姓名、同学姓名、公司内部信息。
- 是否有 API Key、Token、密码、Cookie。
- 是否有 `.env`、本地配置、缓存文件。
- 是否有没打码的截图和附件。

可以让常用智能助手帮你做一次文本扫描：

```text
请帮我检查这批 Markdown 文件是否适合公开上传。
重点找：
1. 姓名、手机号、邮箱、学号、身份证号。
2. 学校、学院、导师、同学、公司等可能暴露身份的信息。
3. API Key、Token、密码、Cookie。
4. 没有脱敏的成绩、排名、申请材料。
请只列风险位置，不要替我编造替换内容。
```

## 推荐仓库结构

把 Obsidian Vault 直接作为仓库，不需要额外构建：

```text
ai-university-knowledge-vault/
├── README.md
├── 使用手册.md
├── 🏠 大学全景图.md
├── 00_快速开始/
├── 10_大学适应与生活/
├── 20_学业规划/
├── 30_职业发展/
├── 40_实验室科研与竞赛/
├── 50_升学与公职/
├── 60_个人成长与资产/
├── 70_核心操作系统/
├── 99_使用手册与进阶/
└── _templates/
```

下载者拿到这个文件夹后，直接用 Obsidian 打开即可。

## 上传前建议加入 .gitignore

`.gitignore`（告诉 Git 哪些文件不要上传的配置文件）可以避免把本地缓存和隐私配置一起传上去。

建议至少包含：

```gitignore
.obsidian/
.DS_Store
.env
*.log
```

如果你希望公开 Obsidian 的部分插件配置，可以单独挑选，但新手不建议这么做。

## 最小上传流程

如果你会用 Git，可以按下面思路：

```bash
git init
git add .
git commit -m "Initial vault release"
git branch -M main
git remote add origin https://github.com/你的用户名/仓库名.git
git push -u origin main
```

如果你不会用命令行，可以直接在 GitHub 页面新建仓库，然后拖拽上传文件夹里的内容。文件多时命令行更稳。

## 上传后检查

上传完成后，打开 GitHub 仓库检查：

- README 是否能正常显示。
- 中文文件名是否正常。
- `🏠 大学全景图.md` 是否存在。
- 文件夹结构是否完整。
- 搜索仓库，确认没有 Token、API Key、手机号等敏感内容。
- 下载 ZIP 后，用 Obsidian 打开，确认链接没有明显丢失。

## 可以让智能助手帮你做什么

```text
我准备把 Obsidian 知识库上传 GitHub。
请帮我做公开前检查：
1. 检查 README 是否说明清楚这是什么、怎么开始、怎么使用。
2. 检查是否有明显隐私字段。
3. 检查 .gitignore 是否覆盖 .obsidian、.env、系统缓存。
4. 检查是否有断开的 Obsidian 双向链接。
5. 输出需要修改的文件列表，不要直接上传。
```
