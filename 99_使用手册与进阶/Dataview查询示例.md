# Dataview查询示例

类型:: 进阶示例
状态:: 可选
阶段:: 基础熟练后
关联目标:: [[进阶插件说明]]
下一步行动:: 先做一个“规划中事项”看板

## Dataview 是干什么的

Dataview（Obsidian 第三方插件，用字段查询笔记）可以把散落在不同文件里的内容自动汇总成表格。

你可以把它理解成一个轻量表格助手：每篇笔记仍然是普通 Markdown，但只要里面写了 `状态:: 规划中`、`阶段:: 大三上`、`下一步行动:: 更新简历` 这样的字段，Dataview 就能把符合条件的笔记找出来。

适合它的场景：

- 汇总所有 `状态:: 规划中` 的事情。
- 找出没有填写 `下一步行动::` 的笔记。
- 列出和 `[[保研准备]]`、`[[考研规划]]`、`[[核心实习计划]]` 有关的材料。
- 做一个“本月要推进什么”的看板。

不适合它的场景：

- 笔记很少，还没有形成稳定字段。
- 想让插件替你判断人生方向。
- 字段名今天写 `阶段`，明天写 `年级`，后天写 `时间段`。

## 安装和打开

1. 打开 Obsidian 设置。
2. 进入“第三方插件”。
3. 关闭安全模式。
4. 搜索 `Dataview` 并安装。
5. 启用插件。

如果你不想自己排查，可以把报错截图或设置页面描述发给常用智能助手，让它按步骤帮你检查。

## 查询 1：所有规划中的事项

适合放在 `我的待办看板.md` 里。

```dataview
TABLE 状态, 阶段, 关联目标, 下一步行动
WHERE 状态 = "规划中"
SORT 阶段 ASC
```

这个表会找出所有写了 `状态:: 规划中` 的笔记。它最适合每周复盘前看一眼，防止很多计划只是写过但没推进。

## 查询 2：缺少下一步行动的笔记

```dataview
TABLE 类型, 状态, 阶段, 关联目标
WHERE !下一步行动
SORT file.name ASC
```

这个表用来抓“写了很多，但不知道下一步做什么”的笔记。看到这类文件，不需要大改，补一句 15 分钟内能做的动作就行。

## 查询 3：大三相关事项

```dataview
TABLE 类型, 状态, 关联目标, 下一步行动
WHERE contains(阶段, "大三")
SORT file.name ASC
```

适合大二下或大三上使用。它能帮你把大三的实习、科研、竞赛、保研、考研准备集中到一张表里。

## 查询 4：所有实习和项目复盘

```dataview
TABLE 状态, 阶段, 技能, 经验沉淀
WHERE contains(file.name, "复盘") OR contains(类型, "复盘")
SORT file.mtime DESC
```

这个表适合求职前看。你可以从里面挑出 3-5 段最有含金量的经历，进一步改写成简历项目、面试故事和作品集说明。

## 查询 5：和升学公职有关的材料

```dataview
TABLE 状态, 阶段, 关联目标, 下一步行动
WHERE contains(关联目标, [[保研准备]]) OR contains(关联目标, [[考研规划]]) OR contains(关联目标, [[考公选调路径]])
SORT file.name ASC
```

如果这条查询没有结果，通常不是 Dataview 坏了，而是相关笔记里没有把 `关联目标::` 写清楚。先补字段，再查。

## 让智能助手帮你写查询

你不需要背 Dataview 语法。更实用的方式是把字段名和目标说清楚：

```text
我在 Obsidian 里使用 Dataview。
我的字段有：
- 类型
- 状态
- 阶段
- 关联目标
- 下一步行动
- 技能
- 经验沉淀

请帮我写 5 个查询，分别用于：
1. 汇总所有规划中的事项。
2. 找出没有下一步行动的笔记。
3. 汇总所有大三相关事项。
4. 汇总实习、竞赛、科研复盘。
5. 汇总和保研、考研、公职有关的材料。

要求：
- 用 dataview 代码块输出。
- 每个查询下面用一句话解释适合什么时候用。
- 不要改我的字段名。
```

## 字段统一很重要

Dataview 不是靠猜，它靠字段。建议这套资料包长期使用下面 5 个基础字段：

- `类型::`
- `状态::`
- `阶段::`
- `关联目标::`
- `下一步行动::`

如果你已经写乱了，可以请智能助手先做一次字段整理：

```text
下面是我知识库里常见的字段名。请帮我合并成一套更稳定的字段规范。
要求：
1. 保留中文字段。
2. 不超过 8 个核心字段。
3. 标出哪些旧字段应该合并到哪个新字段。
4. 输出一份替换建议，不要直接编造内容。
```

## 常见问题

### 表格不显示

先检查三件事：Dataview 是否启用、代码块开头是否写成 `dataview`、字段名是否完全一致。

### 查询结果为空

通常是字段值不匹配。比如查询写 `状态 = "规划中"`，但笔记里写的是 `状态:: 正在规划`，它就不会被找出来。

### 查询太多，页面变慢

先保留 3 个最常用的看板：规划中事项、缺少下一步行动、大三相关事项。其他查询放到需要时再打开。

### 不知道该查什么

从问题倒推，不要从插件倒推。比如你真正的问题是“哪些经历能写进简历”，那查询就应该围绕实习、项目、竞赛、技能、结果证明来写。
