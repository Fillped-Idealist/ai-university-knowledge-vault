# 进阶插件说明

类型:: 插件说明
状态:: 可选
阶段:: 基础熟练后
关联目标:: [[完整操作手册]]
下一步行动:: 只选一个最能减少重复劳动的功能试用

## 先说结论

这套知识库不靠插件也能用。最重要的仍然是：把真实经历写下来，把关键笔记互相连起来，每周做一次复盘。

插件的作用是减少重复劳动，不是增加学习负担。你可以按下面顺序慢慢来：

1. 先用 `[[双向链接]]` 和 `字段:: 值` 跑通基础记录。
2. 笔记超过 20 篇后，考虑用 [[Dataview查询示例]] 做自动清单。
3. 每周都复制同一套格式时，再用 Templater 做模板。
4. 面临复杂选择时，用 [[Canvas使用建议]] 画图。
5. 知识库已经很丰富后，再看 [[MCP进阶连接指南]]。

## Dataview：自动把笔记列成表格

Dataview（Obsidian 第三方插件，用字段查询笔记）适合做看板。比如你想知道：

- 哪些事情 `状态:: 规划中`。
- 哪些项目还没有写复盘。
- 大三下有哪些必须推进的任务。
- 所有和 `[[保研准备]]` 有关的材料在哪里。

最小用法是：先在笔记里统一字段，再把查询语句粘贴到一个看板文件里。字段越稳定，查询越好用。

可以请智能助手帮忙：

```text
请根据我的 Obsidian 字段写 Dataview 查询。
字段包括：类型、状态、阶段、关联目标、下一步行动。
我想要三个表格：
1. 所有规划中的事项。
2. 所有缺少下一步行动的笔记。
3. 所有和升学、公职、实习有关的笔记。
请输出可直接复制的 dataview 代码块。
```

不适合一开始就用 Dataview 的情况：你的笔记还不到 10 篇，字段也没有固定。这个阶段先写内容，别急着做看板。

## Templater：减少重复格式

Templater（Obsidian 第三方插件，用模板自动生成笔记）适合处理反复出现的固定格式，例如：

- 每周复盘。
- 科研周报。
- 实习复盘。
- 面试复盘。
- 竞赛推进记录。
- 读书笔记。

最小用法是：把模板放在 `_templates` 文件夹，在 Obsidian 命令面板里搜索 `Templater: Insert template`，选择模板插入当前文件。

可以请智能助手帮忙：

```text
请把下面这份复盘结构改成 Obsidian 模板。
要求：
1. 保留 字段:: 值。
2. 问题要具体，适合大学生真实复盘。
3. 不要写空泛标题。
4. 输出一份可以保存到 _templates 的 Markdown。
```

如果暂时不想装插件，直接复制 `_templates` 里的文件也可以。Templater 只是让复制更省事。

## Canvas：把复杂关系摆到一张图上

Canvas（Obsidian 自带白板功能）适合处理多路径、多角色、多材料的关系。比如：

- 保研、考研、就业、公职怎么取舍。
- 一个竞赛项目里谁负责什么。
- 毕业论文从选题到答辩的材料链路。
- 一段实习如何连接到技能、简历和作品集。

最小用法是：新建 Canvas，把相关笔记拖进去，用线连接它们。不要在画布里写太多正文，正文仍然放回 Markdown 文件。

可以请智能助手帮忙：

```text
我想做一张大学路径选择 Canvas。
请帮我列出应该放进去的节点、连接关系和每个节点旁边要回答的问题。
节点至少包括：保研准备、考研规划、核心实习计划、考公选调路径、GPA管理、技能树。
请不要替我做最终选择，只帮我把信息结构摆清楚。
```

## Daily Notes：先把事实留住

Daily Notes（Obsidian 自带每日笔记功能）适合记录当天发生的事。它不需要写成文章，只要把事实留下来：

- 今天上了什么课。
- 做了什么项目任务。
- 收到什么反馈。
- 遇到什么问题。
- 有什么值得周末复盘。

建议每天 5 行，周末统一整理。这样比临近申请、求职、考研复试时再回忆可靠得多。

可以请智能助手帮忙：

```text
下面是我这周的每日记录。请整理成周复盘：
1. 只基于已有事实。
2. 分成学业、职业、科研竞赛、生活健康、人际协作。
3. 标出值得更新到长期笔记的内容。
4. 给出下周 3 个具体行动。
```

## 反向链接和图谱：检查关系是不是断了

反向链接（显示哪些笔记提到了当前笔记）和图谱（显示笔记之间的连接）是 Obsidian 自带功能。它们的价值不是“看起来很复杂”，而是帮你发现：

- 重要目标有没有连接到证据。
- 某段经历是否已经沉淀到技能、简历、作品集。
- 有没有关键笔记孤立在角落。

使用时可以重点打开 `[[保研准备]]`、`[[考研规划]]`、`[[技能树]]`、`[[核心实习计划]]` 这些核心节点，看它们是否连接了足够多的材料。

## MCP：让工具直接读你的本地资料包

MCP（模型上下文协议，可以理解为让智能助手连接本地工具和文件的接口）建议最后再学。它适合已经有一批真实笔记的人，用来做跨文件搜索、批量整理、补链接、生成计划。

配置前先确认三件事：

1. 你已经把资料包用了至少 2-4 周。
2. 你清楚哪些内容能公开，哪些只能放私人库。
3. 你愿意先用测试库验证，不直接批量改私人库。

可以请智能助手帮忙：

```text
我想给 Obsidian 资料包配置 MCP。
请先按只读模式帮我检查步骤：
1. 我的资料包路径应该怎么填。
2. 配置文件应该放在哪里。
3. 如何测试能否读取文件。
4. 哪些内容不能写进公开仓库。
请不要直接修改我的私人笔记，先给我确认清单。
```

## 推荐学习顺序

| 阶段 | 先做什么 | 暂时别急 |
| --- | --- | --- |
| 刚打开资料包 | 写第一篇周记，改一篇个人规划 | 不急着装插件 |
| 用了 1 周 | 让智能助手补字段、补链接 | 不急着做复杂看板 |
| 笔记 20 篇以上 | 尝试 Dataview 查询 | 不要一次做十几个表 |
| 重复模板很多 | 使用 Templater | 不要为了自动化改乱模板 |
| 面临路径选择 | 用 Canvas 摆关系 | 不要把所有正文写进画布 |
| 笔记 30 篇以上 | 考虑 MCP 只读连接 | 不要直接开放批量写入 |

## 安全提醒

- 公开仓库只放模板、方法和脱敏案例。
- 私人库才放真实姓名、成绩单、聊天记录、证件、申请材料。
- 任何 API Key、Token、密钥都不要写进 Markdown。
- 批量修改前先备份一份完整资料包。
