# CONTRIBUTING

类型:: 开源说明
状态:: 可参与
关联入口:: [[README]]、[[🏠 大学全景图]]

欢迎改进这个大学生 AI 知识库。贡献时请尽量保持“低门槛、少术语、可复制”的风格。

## 内容原则

- 优先写真实大学场景，不只写工具技巧。
- 每篇笔记尽量包含 `类型::`、`状态::`、`阶段::`、`关联目标::`、`下一步行动::`。
- 使用 `[[双向链接]]` 连接相关笔记。
- 出现术语时用括号解释，例如 MCP（让智能助手连接外部工具的协议）。

## 不建议加入

- 过度复杂的自动化脚本。
- 必须付费才能使用的闭源流程。
- 只适合单一学校的硬编码规则。

## 推荐贡献方向

- 增加不同专业的示例：医学、法学、师范、计算机、商科、艺术。
- 增加真实复盘样例。
- 改进 [[AI使用指令集]]。

## 仓库维护建议

如果你要把这个资料包上传 GitHub，建议保留以下结构：根目录放 README 和使用手册，知识库内容按编号文件夹组织，模板统一放 `_templates`。不要把自己的私人版直接公开；可以从私人版复制空模板和脱敏示例到公开版。

## 适合二次开发的方向

- 专业版：计算机、医学、法学、师范、商科分别扩展专业课程和求职路径。
- 学校版：补充本校教务、奖学金、转专业、保研政策入口。
- 训练营版：把 5 天小项目拆成每日打卡任务。
- 社群版：允许同学提交脱敏案例，形成案例库。

## 模板字段怎么取舍

如果字段太多，用户会不想填；如果字段太少，AI 无法转化。保留能支持未来使用的字段：时间、角色、行动、结果、证据、复盘、下一步。

## 好坏填写对比

低价值写法：参与了一个比赛，负责资料整理，学到了很多。高价值写法：在 X 比赛中负责 30 份用户访谈资料编码，整理出 4 类需求，支持队伍完成商业计划书用户分析部分。

## 模板复用建议

同一个模板可以用于不同场景。比如“项目模板”既能记录课程项目，也能记录竞赛、社团活动、实习任务。关键是保留个人贡献和结果证据。


## 自查标准

读完这一页后，至少应该能判断三件事：什么时候使用它、应该收集哪些信息、下一步该更新哪个文件。如果仍然只知道“去记录一下”，就需要回到真实场景里补事实、证据和例子。
