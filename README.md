# 大学生 AI 个人知识库：把大学经历整理成可复用资产

这是一套面向大学生的 Obsidian 知识库资料包。它不是单纯的笔记模板，也不是工具教程，而是一套围绕大学四年真实场景设计的个人资料系统：入学适应、课程学习、GPA、转专业、社团、学生干部、实验室、科研、竞赛、实习、保研、考研、考公、秋招、毕业论文和个人成长，都有对应的记录位置和整理方法。

很多大学生并不是没有经历，而是经历散在聊天记录、课件、网盘、截图、脑子和临时文档里。等到写简历、保研材料、考研复试、公职面试、毕业论文时，才发现自己说不清过去几年到底积累了什么。这个资料包要解决的就是这件事：把零散经历变成能查找、能复盘、能迁移、能用于申请和求职的长期资产。

## 适合谁

- 刚入学，不知道大学四年该怎么规划的新生。
- 想提高 GPA、准备保研、考研、留学或奖学金申请的同学。
- 正在准备实习、秋招、春招、作品集和面试的同学。
- 想进实验室、做科研、参加竞赛，但不知道怎么沉淀过程材料的同学。
- 想把社团、学生干部、志愿服务、课程项目变成简历和面试素材的同学。
- 不想手动整理大量笔记，希望用更低成本维护个人知识库的同学。

## 你能得到什么

使用一段时间后，你会拥有一个比较完整的大学个人档案库。它能帮你完成这些事：

- 回看每学期课程、成绩、项目和复盘，判断自己的学业状态。
- 整理保研、考研、公职、就业等不同路径所需的证据。
- 把课程项目、社团活动、实习任务、科研竞赛转成简历素材。
- 为个人陈述、推荐信素材、面试故事、毕业总结提供原始材料。
- 通过周记和月度复盘，看见自己反复出现的问题和真实进步。
- 在需要时让智能工具基于你的真实记录做整理、分类、提问和计划。

简单说，这不是让你“多记笔记”，而是让你以后少一点临时抱佛脚。

## 使用说明

这份手册写给第一次打开资料包的人。目标不是让你学一堆工具概念，而是让你在当天就完成第一次部署、第一次记录、第一次整理，并知道后面怎么维护。

你可以把这个资料包理解成一个“大学四年资料柜”：学业、实习、科研、竞赛、升学、公职、社团、个人成长都有固定位置。你平时只要把真实经历放进去，后面写简历、个人陈述、复盘、面试故事时，就不用从零回忆。

## 1. 这个资料包能解决什么问题

### 1.1 没有知识库时，常见问题是什么

大学里的经历经常散在不同地方：

- 课程资料在课件、学习通、网盘、聊天群里。
- 成绩和证书只有截图，申请时临时翻找。
- 社团、竞赛、实习做过不少，但写简历时想不起来细节。
- 保研、考研、就业、公职同时焦虑，但没有证据判断哪条路更适合。
- 每学期都觉得自己很忙，却说不清到底积累了什么。

这套资料包不是让你“多写笔记”，而是帮你把这些零散经历放到固定位置，逐渐形成一套可检索、可复盘、可复用的个人资料系统。

### 1.2 用起来后会得到什么

用一段时间后，你应该能得到这些结果：

- 打开 `GPA管理台`，能看到每学期课程、成绩、风险课程和补救动作。
- 打开 `简历素材库`，能看到社团、课程项目、实习、竞赛如何转成简历素材。
- 打开 `保研准备` 或 `考研规划`，能看到时间线、材料清单、当前证据和缺口。
- 打开 `复盘周记`，能看到每周发生了什么、卡在哪里、下周做什么。
- 打开 `四年经验资产清单`，能把大学经历整理成申请、求职、汇报时可用的材料。

### 1.3 一个具体结果画面

假设你问：我现在应该优先准备考研、就业还是保研？

好的分析不应该只说“看你兴趣”。它应该基于这些文件：

- `GPA管理台`：成绩是否支撑保研或奖学金。
- `科研项目管理`：有没有科研或实验室证据。
- `技能树`：就业所需技能有没有作品支撑。
- `简历素材库`：有没有能投实习或秋招的经历。
- `复盘周记`：你最近真实投入在哪些事情上。

最后给出证据、风险、备选路径和未来 30 天行动。这个结果才有参考价值。

## 2. 安装和环境准备

### 2.1 你需要准备什么

只需要三样东西：

1. Obsidian：本地笔记软件，用来打开这个资料包。
2. 本资料包：也就是这个 GitHub 仓库。
3. 一个常用的智能助手：可以是 Codex、WorkBuddy、CodeBuddy、ChatGPT 或其他能处理 Markdown 的工具。

第一天不需要配置 MCP，也不需要安装 Dataview、Templater、Canvas。先跑通基础流程，再考虑进阶。

### 2.2 下载资料包

在 GitHub 页面下载：

1. 点击绿色 `Code` 按钮。
2. 点击 `Download ZIP`。
3. 解压到一个固定位置，例如：
   - macOS：`Documents/大学AI知识库`
   - Windows：`文档/大学AI知识库`
4. 不建议放在“下载”文件夹，因为后面容易误删或找不到。

如果你熟悉 Git，也可以直接克隆仓库。但新手用 ZIP 就够了。

### 2.3 安装 Obsidian 并打开库

1. 安装 Obsidian。
2. 打开 Obsidian。
3. 选择“打开本地文件夹作为库”。
4. 选择解压后的资料包文件夹。
5. 打开根目录里的 `🏠 大学全景图.md`。

如果你看到学业、职业、科研竞赛、升学公职等入口，说明已经打开成功。

### 2.4 让智能助手帮你完成第一次部署

如果你不想自己判断从哪里开始，可以直接复制下面这段话给常用工具：

```text
我刚下载了一个 Obsidian 大学生知识库资料包。请你作为部署助手，带我完成第一次使用。

我的情况：
- 年级：请问我
- 专业：请问我
- 当前最关心的方向：请问我
- 是否已经安装 Obsidian：请问我

请按以下方式带我操作：
1. 先问我 4 个必要问题，不要一次问太多。
2. 根据我的回答，告诉我应该先打开哪 5 个文件。
3. 帮我生成第一篇“复盘周记”的初稿结构。
4. 告诉我今天只需要更新哪 1-2 个文件。
5. 不要让我一开始配置 MCP、Dataview 或其他插件。
```

它应该先问你年级、专业、当前目标、是否已安装 Obsidian，而不是直接丢给你一堆复杂配置。

### 2.5 隐私准备

不要把这些内容直接发到外部工具或上传到公开仓库：

- 真实姓名、学号、手机号、身份证号、家庭住址。
- 成绩单截图、银行卡信息、宿舍门牌、私人聊天记录。
- 未公开论文数据、实验数据、实习公司内部资料。
- 情绪、人际、财务等只适合自己保留的内容。

推荐做法：公开版只放模板和脱敏案例，私人版才放真实经历。

## 3. 第一次使用：10 分钟完成最小闭环

第一天不要填完整个知识库。只要完成一个闭环：写一段真实记录，整理成笔记，放回 Obsidian。

### 3.1 写一段原始记录

可以很粗糙，例如：

```text
这周刚开学，班级群和学院群消息很多，我有点跟不上。高数第一节课听不太懂。社团招新很多，我想参加但怕影响学习。宿舍作息还没磨合好。
```

### 3.2 用这段提示整理

```text
请把下面这段大学生活记录整理成 Obsidian 知识库笔记建议。

要求：
1. 先判断应该更新哪些文件。
2. 每个文件只给 2-4 句话，不要写长文。
3. 保留真实事实，不要编造学校、成绩、奖项、公司或数据。
4. 如果信息不够，标注“待补充”。
5. 最后告诉我今天只需要更新哪 1-2 个文件。

原始记录：
（把你的记录粘在这里）
```

### 3.3 预期整理结果

好的整理结果大概会这样：

- `入学适应清单`：记录班级群、学院群、通知来源混乱的问题。
- `课程学习系统`：记录高数第一节课听不懂，需要建立课程风险。
- `社团与学生组织`：记录社团招新兴趣和时间担忧。
- `宿舍与人际`：记录宿舍作息磨合问题。
- `复盘周记`：作为本周总入口。

今天只更新 `复盘周记` 和 `课程学习系统` 就够了。

### 3.4 放回 Obsidian

1. 在 Obsidian 打开 `70_核心操作系统/复盘周记.md`。
2. 把整理后的本周记录放进去。
3. 如果涉及高数，再打开 `20_学业规划/课程学习系统.md` 补一句。
4. 不要追求完美格式，先让记录存在。

这就是第一次使用的完整闭环。

## 4. 必须学会的几个基础功能

### 4.1 打开入口

每次不知道从哪里开始，就打开：

```text
🏠 大学全景图.md
```

它相当于整个知识库的地图。

### 4.2 搜索

如果你要找考研，就搜“考研”；要找实习，就搜“实习”；要找周记，就搜“复盘周记”。

建议搜索顺序：

1. 先搜关键词。
2. 再看相关笔记里的双向链接。
3. 最后回到 `🏠 大学全景图.md` 看它属于哪个板块。

### 4.3 双向链接

写法：

```text
[[保研准备]]
```

作用：把当前笔记和 `保研准备.md` 连起来。以后打开保研准备时，就能顺着链接找到 GPA、科研、竞赛、导师沟通、个人陈述等材料。

不用一开始手动补很多链接。你可以先写内容，再统一整理。

### 4.4 字段

写法：

```text
状态:: 规划中
阶段:: 大二上
关联目标:: [[保研准备]]
下一步行动:: 核对学院推免政策
```

字段的作用是让笔记更容易被搜索和整理。最常用字段：

- `类型::` 这是什么笔记。
- `状态::` 规划中、进行中、已完成、待复盘。
- `阶段::` 大一上、大二暑假、大三下等。
- `关联目标::` 它服务哪个目标。
- `下一步行动::` 最近能做什么。

### 4.5 模板

`_templates` 文件夹里有常用模板。使用方式：

1. 打开模板。
2. 复制内容。
3. 新建一篇自己的笔记。
4. 粘贴模板。
5. 只填能填的部分。

最常用模板：

- `实习复盘模板.md`
- `课程项目模板.md`
- `科研周报模板.md`
- `竞赛项目模板.md`
- `面试复盘模板.md`
- `决策平衡单模板.md`

### 4.6 常用整理指令

打开：

```text
70_核心操作系统/AI使用指令集.md
```

里面的内容可以直接复制。新手最常用的是三类：

- 整理零散记录。
- 补字段和双向链接。
- 把经历改写成简历或复盘素材。

## 5. 5 天小项目：搭建个人版大学知识库

每天 15-20 分钟。目标不是填完整个系统，而是让它开始服务你的真实生活。

### 第 1 天：建立入口和第一篇周记

打开：

- `🏠 大学全景图.md`
- `70_核心操作系统/复盘周记.md`

做三件事：

1. 写下年级、专业、当前最关心的目标。
2. 写下最近一周最值得记录的 3 件事。
3. 让工具帮你判断应该更新哪些文件。

验收标准：你至少有一篇周记，里面有真实事件，而不是空模板。

### 第 2 天：搭建学业系统

打开：

- `20_学业规划/GPA管理台.md`
- `20_学业规划/课程学习系统.md`
- `20_学业规划/期末复习计划.md`

填写：

- 本学期课程。
- 学分高或最担心的课程。
- 最近一次作业、小测或课堂反馈。
- 下一周最小复习动作。

可复制提示：

```text
请根据我的课程记录，帮我判断本学期最需要关注的 3 门课。请按学分、难度、当前掌握程度排序，并给出未来 7 天行动。没有信息的地方写待补充。
```

验收标准：你能说出本学期最重要的 1-3 门课，以及为什么。

### 第 3 天：搭建经历素材库

打开：

- `30_职业发展/简历素材库.md`
- `30_职业发展/技能树.md`
- `_templates/简历经历模板.md`

填写 3 条经历：

- 一条课程项目。
- 一条社团/学生工作/志愿服务。
- 一条实习/兼职/竞赛/科研经历，如果没有就写“待补充”。

每条经历至少写：我做了什么、结果是什么、能证明什么能力。

验收标准：你至少有 3 条以后能继续加工的经历素材。

### 第 4 天：比较升学、就业、公职路径

打开：

- `50_升学与公职/保研准备.md`
- `50_升学与公职/考研规划.md`
- `30_职业发展/核心实习计划.md`
- `50_升学与公职/考公选调路径.md`
- `70_核心操作系统/决策平衡单.md`

每条路只填两项：

- 我为什么考虑它。
- 我现在有什么证据支持它。

可复制提示：

```text
请根据我目前填写的保研、考研、就业、公职四条路径，帮我做一张对比表。维度包括：当前证据、最大缺口、时间成本、最小验证行动。不要替我做最终决定。
```

验收标准：你不一定做出选择，但要知道下一步应该验证什么。

### 第 5 天：建立维护节奏

打开：

- `70_核心操作系统/复盘周记.md`
- `70_核心操作系统/行动清单.md`
- `60_个人成长与资产/四年经验资产清单.md`

做三件事：

1. 把本周新增内容汇总到周记。
2. 列出下周 3 个行动。
3. 标出哪些内容以后可以进入简历、保研材料或复盘报告。

验收标准：你知道下周继续更新哪 1-2 个文件。

## 6. 让智能助手帮你快速部署

如果你希望尽量少手动操作，可以按下面的流程来。

### 6.1 部署助手提示词

```text
请你作为我的大学知识库部署助手，带我把这个 Obsidian 资料包用起来。

请按这个流程执行：
1. 先确认我是否已经下载资料包、安装 Obsidian、打开 Vault。
2. 如果我没有完成，请一步一步告诉我怎么做，每次只给一个动作。
3. 完成打开后，请根据我的年级、专业和当前目标，推荐最先看的 5 个文件。
4. 请帮我生成第一篇复盘周记，不要超过 300 字。
5. 请告诉我今天只需要更新哪 1-2 个文件。
6. 不要让我一开始配置 MCP、Dataview、Templater 或 Canvas。
7. 如果我提供的信息不够，请先追问，不要编造。
```

### 6.2 文件整理提示词

```text
我会给你一段零散记录。请帮我整理成适合放进 Obsidian 知识库的内容。

输出格式：
1. 应该更新的文件名。
2. 每个文件建议写入的内容。
3. 需要补充的证据。
4. 相关双向链接。
5. 今天最小行动。

限制：不编造学校、成绩、奖项、公司、论文、证书、数据。没有证据就写待补充。
```

### 6.3 每周维护提示词

```text
下面是我本周的记录。请帮我整理成周复盘，并告诉我哪些内容应该更新到学业、职业、科研竞赛、升学公职、个人成长板块。

要求：
- 先总结本周 3 个事实。
- 再列出 3 个风险或重复问题。
- 再列出应该更新的文件。
- 最后给出下周 3 个最小行动。
```

### 6.4 判断部署是否成功

满足下面 5 条，就算部署成功：

- Obsidian 能打开资料包。
- `🏠 大学全景图.md` 能正常打开。
- `复盘周记.md` 里已经有一段真实记录。
- 至少 1 个学业或职业文件被更新过。
- 你知道下次继续从哪个文件开始。

## 7. 进阶功能怎么用

这一节不是让你一次学完所有插件。更现实的做法是：先用普通笔记跑起来，发现某个重复动作很烦，再挑一个功能解决它。下面每个功能都按“它能做什么、第一步怎么用、怎么请智能助手帮忙”来写。

### 7.1 Dataview：把笔记自动汇总成清单

Dataview（Obsidian 插件，用来按字段自动整理笔记）适合解决一个问题：笔记越来越多以后，你不想一篇篇翻，只想看到“哪些事情还在规划中”“哪些项目缺复盘”“大三下有哪些关键任务”。

最小用法：

1. 在 Obsidian 左侧点“设置”。
2. 打开“第三方插件”，关闭安全模式后搜索并安装 `Dataview`。
3. 在一篇新笔记里粘贴查询语句，让它自动列出符合条件的文件。

例如，你可以新建一篇 `我的待办看板.md`，放入：

```dataview
TABLE 状态, 阶段, 下一步行动
WHERE 状态 = "规划中"
SORT file.name ASC
```

这段话的意思很简单：把所有 `状态:: 规划中` 的笔记列成表格，并显示阶段和下一步行动。

可以请智能助手帮你的地方：

```text
我想在 Obsidian 里用 Dataview 做一个看板。
我的笔记字段有：类型、状态、阶段、关联目标、下一步行动。
请帮我写 3 条查询：
1. 所有状态为“规划中”的事项。
2. 所有阶段包含“大三”的事项。
3. 所有下一步行动为空的笔记。
请用适合直接复制到 Obsidian 的代码块输出。
```

注意：Dataview 只会整理你已经写进笔记里的字段。字段太乱时，先让智能助手帮你统一字段名，比硬写查询更省事。

### 7.2 Templater：让重复模板自动填好

Templater（Obsidian 插件，用来自动生成固定格式笔记）适合减少重复劳动。比如周记、科研周报、实习复盘、面试复盘，每次都要复制同一套标题和字段，用久了会烦。

最小用法：

1. 安装 `Templater` 插件。
2. 在设置里把模板文件夹指向 `_templates`。
3. 打开命令面板（Mac 通常是 `Command + P`，Windows 通常是 `Ctrl + P`），搜索 `Templater: Insert template`。
4. 选择一个模板插入到当前笔记。

适合做成模板的内容：

- `复盘周记.md`：每周固定写一次。
- `实习复盘模板.md`：每段实习、项目、兼职都能套用。
- `科研周报模板.md`：进实验室后每周记录进度。
- `面试复盘模板.md`：每次面试后立刻沉淀题目和反馈。

可以请智能助手帮你的地方：

```text
请把下面这篇笔记改造成 Obsidian 的 Templater 模板。
要求：
1. 保留 字段:: 值 的结构。
2. 日期位置留出自动日期或手动填写的位置。
3. 每个问题都要具体，不要写“总结一下”这种空泛标题。
4. 模板适合大学生每周复盘使用。
```

如果你暂时不想学 Templater，也没关系。直接复制 `_templates` 里的模板文件，同样能用。

### 7.3 Canvas：把复杂选择画成一张图

Canvas（Obsidian 自带的白板功能，用来摆放笔记和连接关系）适合处理“光看文字容易乱”的问题。比如保研、考研、就业、公职四条路同时存在时，你需要看到它们各自的条件、成本、风险和下一步。

最小用法：

1. 在 Obsidian 新建一个 Canvas 文件。
2. 把 `[[保研准备]]`、`[[考研规划]]`、`[[核心实习计划]]`、`[[考公选调路径]]` 拖进去。
3. 给每条路径旁边加一个小卡片，写“优势、风险、下一步证据”。
4. 用线连接相关笔记，比如 `[[GPA管理台]]` 同时影响保研和留学，`[[核心实习计划]]` 影响就业和作品集。

一个好用的画布结构：

- 中心：`大学四年主线选择`
- 左侧：已有条件，放 `GPA`、证书、项目、竞赛、科研、实习。
- 右侧：目标路径，放保研、考研、就业、公职、留学。
- 下方：当前缺口，放英语、论文、项目证明、推荐人、作品集。
- 顶部：最近 30 天行动，放最紧急的 3 件事。

可以请智能助手帮你的地方：

```text
我想用 Obsidian Canvas 画一张“保研、考研、就业、公职”对比图。
请根据下面的个人情况，帮我列出：
1. 应该放进画布的节点。
2. 节点之间应该怎么连接。
3. 每条路径旁边应该写哪些判断问题。
4. 最近 30 天最该补的证据。
请不要替我做结论，只帮我把信息摆清楚。
```

Canvas 不适合替代正文笔记。它更像地图：帮你看清关系，具体材料还是放回对应 Markdown 文件里。

### 7.4 Daily Notes 和模板：把日常记录留住

Daily Notes（Obsidian 自带的每日笔记功能）适合记录当天发生的事。它不要求你写得漂亮，只要把事实留住：上了什么课、做了什么项目、参加了什么活动、收到什么反馈、遇到什么问题。

最小用法：

1. 在 Obsidian 设置里打开“核心插件”。
2. 启用 `Daily Notes`。
3. 把每日笔记位置设成 `60_个人成长与资产/每日记录` 或你自己喜欢的位置。
4. 每天只写 5 行事实，周末再整理。

可以请智能助手帮你的地方：

```text
下面是我这周的每日记录。请帮我整理成一篇周复盘：
1. 只基于我写过的事实，不要编。
2. 把内容分到学业、职业、科研竞赛、生活健康、人际协作。
3. 找出值得更新到长期笔记的内容。
4. 最后给我下周 3 个具体行动。
```

这套资料包最推荐的节奏是：每天随手记，周末做复盘，月底整理成项目、课程、技能或申请材料。

### 7.5 反向链接和图谱：找到被忽略的关系

反向链接（显示哪些笔记提到了当前笔记）和图谱（把笔记关系画成网络）不需要额外安装。它们适合用来检查一个问题：这篇笔记是不是孤立的。

简单用法：

- 打开 `[[技能树]]`，看哪些课程、项目、实习提到了它。
- 打开 `[[保研准备]]`，看它是否连接了 GPA、科研、竞赛、推荐人、材料清单。
- 打开图谱视图，看看有没有完全孤立的文件。孤立不一定错，但关键文件不应该孤立。

可以请智能助手帮你的地方：

```text
请检查下面这篇笔记应该连接到哪些已有笔记。
要求：
1. 推荐 5-8 个 [[双向链接]]。
2. 每个链接说明为什么有关。
3. 不要为了凑数量硬连。
4. 如果缺少应该新建的笔记，也请列出来。
```

### 7.6 MCP：让常用工具直接读取资料包

MCP（模型上下文协议，可以理解为让智能助手连接本地工具和文件的接口）适合放在最后学。它的价值是：当你的知识库已经有几十篇真实笔记后，不用再反复复制粘贴，工具可以在你授权后读取资料包，帮你跨文件查找、整理、补链接、生成计划。

建议满足这些条件再配置：

- 你已经稳定用了 2-4 周。
- 知识库里有 30 篇以上真实笔记。
- 你经常需要跨很多文件做整理。
- 你知道哪些内容不能给外部工具读取。

配置原则：

1. 先复制一份测试库，不要直接拿私人库试。
2. 先只读，确认能正常搜索文件后，再考虑写入。
3. 不要把 API Key、Token、密钥、身份证号、完整成绩单、聊天记录上传 GitHub。
4. 任何批量修改前先备份，最好先让工具列计划，再确认执行。

可以请智能助手帮你的地方：

```text
我想把 Obsidian 资料包连接到常用智能助手。
请先不要直接改文件，只帮我做三件事：
1. 检查我的资料包路径是否正确。
2. 给出只读模式的配置步骤。
3. 列出配置后可以测试的 5 个问题。
如果涉及密钥或 Token，请提醒我用环境变量，不要写进公开文件。
```

MCP 不是入门必需品。它更像后期加速器：基础记录越真实，它发挥的作用越大；前期内容太少时，先把周记和核心经历补起来更划算。

## 8. 常见问题

### 打开后文件太多，不知道从哪开始

只看三个文件：`🏠 大学全景图.md`、`复盘周记.md`、`AI使用指令集.md`。其他文件以后再慢慢用。

### 不知道该填哪些字段

先填 5 个：`类型::`、`状态::`、`阶段::`、`关联目标::`、`下一步行动::`。填不准也没关系，后面可以改。

### 整理结果太空泛

通常是原始记录缺少事实。把“我参加了比赛”补成“我参加了什么比赛、负责什么、产出什么、结果是什么、证据在哪里”。

### 担心隐私

先用脱敏版本。比如把学校写成“某双非/某 985/某师范院校”，把公司写成“某互联网公司”，把同学写成“同学A”。

### 可以不用 Obsidian 吗

可以，因为内容都是 Markdown 文件。但 Obsidian 的双向链接、搜索和图谱更适合这套资料包。

## 9. 最后记住这四句话

1. 先记录真实经历，不追求完美格式。
2. 先跑通周记，再扩展其他板块。
3. 先用复制整理，后学插件和 MCP。
4. 每周复盘一次，让大学经历持续变成资产。
